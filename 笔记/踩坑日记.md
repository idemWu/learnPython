### 坑 1：Pydantic 对象不能直接当字典用

```python
from pydantic import BaseModel

class UserCreate(BaseModel):
    name: str
    age: int

# 错误：user 是 Pydantic 模型实例，不是普通字典
fake_db.append(user)

# 正确：需要转成字典
fake_db.append(user.model_dump())
```

Pydantic 对象虽然看起来像字典（有 `.name`、`.age` 属性），但它是一个**类实例**，不能直接 `append` 到列表里当 JSON 返回。需要用 `.model_dump()` 转成普通字典。

### 坑 2：`uvicorn.run()` 忘记加 `reload`

```python
# 漏了 reload，代码改了不会自动重启
uvicorn.run(app, host='127.0.0.1', port=8000)

# 加上 reload，开发体验好很多
uvicorn.run(app, host='127.0.0.1', port=8000, reload=True)
```

> 注意：`python main.py` 启动时 `reload` 有时会出问题（主模块重载），所以推荐用命令行方式 `uvicorn main:app --reload`。

### 坑 3：`BaseModel` 拼写错误，多了个 `s`

```python
# 错误
from pydantic import BaseModels

# 正确
from pydantic import BaseModel
```

类名是 `BaseModel`（单数），不是 `BaseModels`。

### 坑 4：路径参数不声明类型，匹配失败

```python
# 错误：user_id 是字符串，和数字 1 比较永远不等
@app.get("/users/{user_id}")
def search_user(user_id):
    if u["id"] == user_id:  # "1" == 1 → False
        ...

# 正确：加上 : int，FastAPI 自动转换
def search_user(user_id: int):
    if u["id"] == user_id:  # 1 == 1 → True
        ...
```

### 坑 5：`passlib` + 新版 `bcrypt` 不兼容

新版 `bcrypt`（5.x）删掉了 `__about__` 属性，`passlib` 依赖它检测版本，导致密码加密时报错：

```
ValueError: password cannot be longer than 72 bytes
```

修复：
```bash
pip install bcrypt==4.0.1
```

### 坑 6：`.env` 提交到 GitHub 后，如何停止跟踪（保留本地文件）

场景：已经把 `.env`（或 `.venv`）传到了远程仓库，但现在想让 Git 停止跟踪它们，同时不删除本地文件。

处理命令：

```bash
# 1) 先在 .gitignore 里加上（避免下次再次被跟踪）
.env
.venv/

# 2) 从 Git 跟踪中移除（只移除索引，不删本地文件）
git rm --cached .env
git rm -r --cached .venv

# 3) 提交并推送到当前分支
git add .gitignore
git commit -m "stop tracking .env and .venv"
git push

# 4) 验证是否还被跟踪（理想情况无输出）
git ls-files | findstr /i ".env .venv"
```

说明：
- 若 `git rm -r --cached .venv` 报 `pathspec ... did not match`，表示 `.venv` 本来就没被跟踪，可忽略。
- `.gitignore` 只能阻止“未来再次被跟踪”，对已跟踪文件必须先执行 `git rm --cached`。
- 如果 `.env` 里有密钥，建议立即去对应平台重置。
